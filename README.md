## 1. 시스템 개요

| 항목 | 내용 |
|------|------|
| 파일명 | `Chungbuk_Traffic_Bot.py` |
| 언어 | Python 3.9 |
| 목적 | 충청북도 지역 교통 관련 뉴스를 자동 수집하여 텔레그램으로 전송 |
| 실행 환경 | GitHub Actions (스케줄 기반, 매일 KST 08:30) 또는 로컬 실행 |

## 4. 함수별 상세 설계

### 4.1 `is_recent_news(pub_date_str)` — 최신 기사 여부 확인

**목적:** 기사 발행일이 현재 시각으로부터 24시간 이내인지 확인하여 오래된 기사를 제외

**처리 흐름:**
1. `email.utils.parsedate_to_datetime()`으로 RFC 2822 형식의 날짜 문자열 파싱
2. 현재 시각과의 차이(`timedelta`)를 계산
3. 차이가 1일 미만이면 `True`, 아니면 `False` 반환
4. 파싱 실패 시 예외를 포착하고 `False` 반환

**반환값:** `bool`

---

### 4.2 `is_valid_news(title)` — 뉴스 유효성 필터링

**목적:** 교통과 무관한 기사를 제거하고, 교통사고·행정·정책·인프라 뉴스를 폭넓게 선별

**처리 흐름:**

```
입력(기사 제목)
    │
    ▼
[1단계] 블랙리스트 검사 ──── 포함 시 ──▶ False (차단)
    │
    │ 미포함
    ▼
[2단계] 화이트리스트 검사 ── 포함 시 ──▶ True (통과)
    │
    │ 미포함
    ▼
  False (차단)
```

**블랙리스트 (8개 — 교통과 무관한 최소한의 차단):**

| 범주 | 키워드 |
|------|--------|
| 범죄 | 살인, 마약, 성폭행, 보이스피싱 |
| 부동산/스포츠 | 기획부동산, 아파트 분양, 프로야구, K리그 |

**화이트리스트 (14개 — 교통 종합 키워드):**

```
교통사고, 추돌, 정체, 통제, 공사,
도로교통공단, 무인단속, 단속장비,
스쿨존, 어린이보호구역, 신호체계,
교통안전, 교차로, 통행제한
```

**판정 로직:** 블랙리스트 우선 적용 → 화이트리스트 매칭 순서로 동작

---

### 4.3 `get_news_score(item)` — 기사 신뢰도·중요도 점수 산정

**목적:** 중복 제거 시 살릴 기사를 결정하고, 텔레그램 발송 순위를 정하기 위한 가점 로직

**채점 기준:**

| 항목 | 조건 | 점수 |
|------|------|------|
| 네이버 뉴스 in-link | `link`에 `n.news.naver.com` 포함 | +10점 |
| 제목 길이 | 30자 초과 | +5점 |
| 지역 키워드 | 제목에 충북 12개 시·군 중 1개 이상 포함 | +5점 |
| 행정/인프라 키워드 | 제목에 인프라 키워드 1개 이상 포함 | +5점 |

**지역 키워드 목록 (12개):**

```
충북, 청주, 충주, 제천, 보은, 옥천,
영동, 증평, 진천, 괴산, 음성, 단양
```

**행정/인프라 키워드 목록 (5개):**

```
도로교통공단, 무인단속, 스쿨존, 어린이보호구역, 교통안전
```

**반환값:** `int` (점수가 높을수록 우선순위 높음, 최대 25점)

---

### 4.4 `extract_nouns(text)` — Kiwi 형태소 분석 기반 명사 추출

**목적:** 기사 제목에서 의미 있는 형태소(명사·숫자)만 추출하여 TF-IDF 벡터화용 코퍼스를 생성

**처리 흐름:**
1. `kiwipiepy.Kiwi.tokenize()`로 입력 텍스트를 형태소 분석
2. 품사 태그가 아래 3종에 해당하는 형태소만 선별:
   - `NNG` (일반명사): 도로, 교통, 사고 등
   - `NNP` (고유명사): 청주, 충북, 한국도로공사 등
   - `SN` (숫자): 사상자 수, 도로 번호 등 기사 식별 핵심 단서
3. 선별된 형태소를 띄어쓰기로 연결하여 코퍼스 문자열 반환

**반환값:** `str` (예: `"청주 도로 추돌 사고 3 부상"`)

---

### 4.5 `deduplicate_news(news_list)` — NLP 기반 중복 제거

**목적:** 보도자료 특성상 문구만 약간 다른 중복 기사를 TF-IDF + 코사인 유사도로 정밀 탐지하여 제거

**알고리즘:**

```
[점수 내림차순 정렬된 기사 리스트]
    │
    ▼
[1] 각 기사 제목 → extract_nouns() → 명사 코퍼스 생성
    │
    ▼
[2] 빈 코퍼스 방어: 명사 0개인 문서는 "_empty_" 더미 토큰으로 대체
    │
    ▼
[3] TfidfVectorizer로 코퍼스 벡터화 → TF-IDF 행렬 생성
    │
    ▼
[4] cosine_similarity()로 N×N 유사도 행렬 계산
    │
    ▼
[5] 유사도 > 0.55 (55%)인 쌍 탐지 → 점수가 낮은 기사를 제거 대상에 추가
    │
    ▼
[6] 제거 대상을 뺀 최종 리스트 반환
```

**핵심 특징:**
- 기존 `difflib.SequenceMatcher`(문자 순서 비교) 대비, **단어 의미 기반 비교**로 보도자료 중복 탐지 정확도 향상
- 숫자(SN) 포함으로 "3중 추돌"과 "5중 추돌"을 다른 사건으로 구분 가능
- `ValueError` 방어: 모든 문서의 vocabulary가 비는 극단적 케이스에서도 안전하게 원본 리스트 반환
- 시간 복잡도: O(n²) (n = 수집된 기사 수, 최대 100건이므로 실용적)

### 4.6 `fetch_traffic_news()` — 뉴스 수집 및 정제

**목적:** 네이버 뉴스 검색 API를 호출하여 충북 교통 뉴스를 수집·정제·점수 정렬·중복 제거

**검색 쿼리 목록 (5개):**

| # | 검색어 |
|---|--------|
| 1 | 충북 교통 사고 |
| 2 | 청주 도로 통제 |
| 3 | 충북 도로공사 |
| 4 | 충북 실시간 교통 |
| 5 | 충북 교통 정체 |

**API 호출 상세:**

| 항목 | 값 |
|------|-----|
| 엔드포인트 | `https://openapi.naver.com/v1/search/news.json` |
| HTTP 메서드 | GET |
| 정렬 기준 | `sim` (유사도순) |
| 요청 건수 | 쿼리당 20건 (`display=20`) |
| 최대 수집량 | 5쿼리 × 20건 = 100건 (이론상 최대) |
| 인증 헤더 | `X-Naver-Client-Id`, `X-Naver-Client-Secret` |

**처리 파이프라인:**

```
[API 응답 수신]
    │
    ▼
[HTML 태그 제거]       <b>, </b>, &quot;, &apos; 치환
    │
    ▼
[is_recent_news]      발행 24시간 이내 기사만 통과
    │
    ▼
[is_valid_news]       블랙리스트/화이트리스트 필터링
    │
    ▼
[get_news_score]      각 기사에 신뢰도·중요도 점수 부여
    │
    ▼
[점수 기준 내림차순 정렬]
    │
    ▼
[deduplicate_news]    Kiwi + TF-IDF + 코사인 유사도 기반 중복 제거
    │
    ▼
[반환]  list[dict] — [{"title": str, "link": str, "score": int}, ...]
```

**HTML 엔티티 치환 규칙:**

| 원본 | 치환 |
|------|------|
| `<b>` | (제거) |
| `</b>` | (제거) |
| `&quot;` | `"` |
| `&apos;` | `'` |

### 4.7 `send_telegram(news_list)` — 텔레그램 메시지 전송

**목적:** 정제된 뉴스 리스트를 텔레그램 메시지로 포맷팅하여 전송

**API 호출 상세:**

| 항목 | 값 |
|------|-----|
| 엔드포인트 | `https://api.telegram.org/bot{TOKEN}/sendMessage` |
| HTTP 메서드 | POST |
| 파라미터 | `chat_id`, `text`, `disable_web_page_preview=True` |

**메시지 포맷:**

뉴스가 있는 경우:
```
🚗 [2026년 04월 01일 충북 교통 뉴스 브리핑]

1. 기사 제목
🔗 기사 링크

2. 기사 제목
🔗 기사 링크

...

💡 24시간 이내 최신 뉴스 중 신뢰도가 높은 기사를 엄선했습니다.
```

뉴스가 없는 경우:
```
📢 2026년 04월 01일
오늘 충북 지역의 신규 교통 뉴스가 없습니다.
```

**제약 사항:**
- 최대 전송 기사 수: **10건** (`news_list[:10]`)
- 링크 미리보기: **비활성화** (`disable_web_page_preview: True`)

## 7. 기술적 특성 및 제약 사항

| 항목 | 상세 |
|------|------|
| 에러 처리 | 메인 함수에 try-except 1단계 적용. API 개별 호출 실패 시 해당 쿼리 건너뜀 (status_code 200 검사) |
| 재시도 로직 | 없음 (API 호출 실패 시 해당 쿼리 결과 누락) |
| 로깅 | `print` 기반 콘솔 출력만 사용 |
| 데이터 저장 | 없음 (메모리 내 처리 후 즉시 전송, 상태 비저장) |
| 실행 방식 | 1회 실행 후 종료 (데몬/스케줄러 아님) |
| 스케줄링 | GitHub Actions cron 등 외부 스케줄러에 의존 |
| 중복 판정 기준 | Kiwi 형태소 분석 + TF-IDF 코사인 유사도 0.55 (55%) 초과 시 중복 간주 |
| 텔레그램 메시지 제한 | 텔레그램 메시지 최대 길이(4096자) 초과 시 별도 처리 없음 |

## 8. 외부 API 의존성 정리

### 8.1 네이버 검색 API

- **문서:** https://developers.naver.com/docs/serviceapi/search/news/news.md
- **인증:** 클라이언트 ID + Secret (헤더 방식)
- **일일 호출 한도:** 25,000건/일 (기본)
- **본 봇 1회 실행당 호출 수:** 5회

### 8.2 텔레그램 Bot API

- **문서:** https://core.telegram.org/bots/api#sendmessage
- **인증:** Bot Token (URL 경로 포함)
- **본 봇 1회 실행당 호출 수:** 1회

## 9. 배포 환경 (GitHub Actions)

```yaml
# .github/workflows/daily_run.yml
name: Daily Chungbuk Traffic News

on:
  schedule:
    # KST 오전 08:30 = UTC 전날 23:30
    - cron: '30 23 * * *'
  workflow_dispatch:  # 수동 실행 버튼

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'

      - name: Install dependencies
        run: pip install requests kiwipiepy scikit-learn

      - name: Run news bot
        env:
          NAVER_ID: ${{ secrets.NAVER_ID }}
          NAVER_SECRET: ${{ secrets.NAVER_SECRET }}
          TELEGRAM_TOKEN: ${{ secrets.TELEGRAM_TOKEN }}
          CHAT_ID: ${{ secrets.CHAT_ID }}
        run: python Chungbuk_Traffic_Bot.py
```

|------|-----|
| 실행 시각 | 매일 KST 08:30 (UTC 23:30) |
| Python 버전 | 3.9 |
| 실행 환경 | ubuntu-latest |
| 수동 실행 | `workflow_dispatch` 지원 |
